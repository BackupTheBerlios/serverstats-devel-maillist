> PS: Gibt es für die Änderung einen konkreten Anlass?

Naja, ich wurde mal von jemandem gefragt, ob er das in einem privaten
Projekt verwenden kann. Außerdem hab ich inzwischen selbst die Erfahrung
gemacht, dass die GPL für Bibliotheken für eigene Projekte etwas
restriktiv ist. Problem hierbei ist auch, dass nicht definiert wird, ob
die Möglichkeit öffentlich über einen Webserver auf ein Programm
zuzugreifen, dass GPL-Code verwendet, die Verbreitung von Binärcode
und/oder abgeleitetem Code darstellt. Der dazugehörige GPL-FAQ-Eintrag
liest sich zwar so, dass es erlaubt ist, allerdings wird darauf
hingewiesen, dass man hier kaum von "privater Nutzung" sprechen kann. Es
ist also eine Grauzone und ich könnte mir vorstellen, dass man bei einer
Anklage deswegen Recht zugesprochen bekommen könnte (das es keine
"privater Nutzung" sein kann). Insofern finde ich gerade für
Webbiliotheken die GPL etwas unpassend.
Da die LGPL trotzdem definiert, dass der Code frei bleiben muss und nur
die Einschränkung entfernt, die das Einbinden in nicht-GPL-Projekte
verhindert, sehe ich auch keinen großen Verlust an dem Gedanken, der
hinter der Wahl der GPL stand. Außerdem bietet die LGPL die Möglichkeit
den Code ohne weiteres wieder unter die GPL zu stellen (Lizenzwechsel
zur GPL ist laut der LGPL explizit erlaubt). Somit kann der Schritt
jederzeit problemlos rückgängig gemacht werden. Der alte Code bleibt
hierbei natürlich trotzdem unter der LGPL verfügbar, insofern ihn
irgendwer noch zum Download anbietet.
