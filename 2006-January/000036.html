> I've got few ideas for your project :

I CC this email to the devel-mailinglist ;-)

> - Configuration using xml file (simple xml should be a good choice ?)

Whats the advantage of this?
With the current configuration-style this is not possible
(classes-instances are created inside the config).

> - An administration interface (W3C XFORMS standard might be a good
> choise to update XML config file)

Thats somewhere on my todo. But I don't think an editor can ever feature
all things possible with the current config-format.
Besides I think an editor should be developed independent to
Serverstats. It could be an additional tool you can download.

> - It would be interestant to have an XML rather than XHTML.  Because If
> you use an XSL stylesheet in conjunction with XML file we can output to
> RSS feed or XHTML file or any other format ...

Thats a nice idea. :)
But as the current format is XML (XHTML is an XML-format) you can do
this with the current files by now.

An RSS-feed would only be nice only if it includes the current values I
think (any other suggestions?). This could be done using an additional
script. I don't see a reason why the current value should be included
somewhere in the normal output.

> -  A sort of SOAP/REST server to allow to get data from other servers

It is planned to use SNMP in the future.

Of course it is possible to create an SOAP/REST-source, that loads the
data from somewhere in the net. On the server-side a simple script that
plots all values from one source to a nice XML-file should be enough, so
the server does not need to have the whole Serverstats installed.
