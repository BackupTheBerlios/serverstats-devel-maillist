> When I've made  the xml config by hosts approach but in reality we
> should have  by groups of sources approach.

I have groups by graph, because sources could poll multiple
hosts/services/whatsoever. A graph is always linked to one (or more)
category.
And, of course, it's much easier to sort graphs directly, because a
graph could contain multiple sources from multiple hosts.

> For the current job, that I currently working on  I require this and
> templates support
> So I can use tree  to  groups my source :D

I think with XML-support you can sort your source-*files* any way you
want, because you put them together using xincludes (or XSLT). So I
don't see any reason to have some structure inside that sources.
